## First Mini CKB Project:
- On-Chain CKB Messaging App

### Phase 2:
- Implementation and Testing

### Objective:
- To create a decentralized application that stores messages (data) on the Nervos blockchain system. The project covers key concepts: Cells, Data field, Transactions, Capacity Usage.

### What i did
- Connected my project to the CKB devenet
- Created two lib files for the core logic of the app
- The first file (store-message.ts) performs four functions
    - (textToHex(text)) - This converts the text the user inputs to hex 
    - (hexToText(hex)) - This reverses the textToHex for reading the messages back 
    - (storeMessage(message)) - This store the message and sends the transaction to the CKB network
- Then I added a fetchMessages() to read all messages stored on chain
- Then I created two component files for the UI 
    - The first file (MessageForm.tsx) takes/handles users input and sends it to the storeMessage() triggering it
    - The second file (MessageList.tsx) displays all messages from the blockchain
- Fixed the UI design 
- Tested the the project 
- Documented all steps and code snippets

### What i learnt
- For good practices, always seperate the blockchain logic from the UI. UI files should never talk to the blockchain directly.
- CKB stores data as hex bytes, not plain text
- How hex conversion works
- The Data field is where messages are stored 
- Messages are permanent, once stored, anyone can see it forever
- Only the signer can modify a cell using his lockscript
