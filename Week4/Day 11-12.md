### Today's Task
- Look into how CKB's verification model differs from EVM

### EVM (Ethereum Virtual Machine)
The Ethereum blockchain uses an execution model. When you make a transaction, the smart contract executes, changing the state of the blockchain. It is in full control. It determines what actions take place, which parties get to perform some specific actions, and what the state will look like after the action. Your balance is located INSIDE the contract, not your wallet.

### CKB-VM — Validation Model
CKB, on the other hand, uses a validation model. Rather than executing state-changing code, CKB executes code that verifies if the transaction is valid.
Each individual cell has its own Lock Script. Whenever an attempt is made to unlock a cell, the Lock Script gets executed and makes sure, "Is this person entitled to unlock me?" In case the request is true, the response is zero; otherwise, it gives a non-zero output and rejects the transaction.

### The 3 biggest differences I noticed
1. The Execution vs the Validation:
In the EVM, code runs and changes state, while in CKB, code runs too, but it also validates proofs.

2. Where each state lives:
EVM states stay centralized inside the smart contract, while the CKB state is of decentralized ownership and stays inside individual cells. 

3. Who controls it:
For EVM, the smart contract controls your assets, while for CKB, you have control of your own assets through your own cells.
