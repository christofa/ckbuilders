## Second  Mini CKB Project:
- CKB Token dApp

### Phase 1:
- Project Setup and Understanding 

### What the App does:
- Allows users generate token (sUDT)
- Determine supply of the token 
- Keep the tokens in on-chain CKB cells
- Transfer tokens fron one user to another 
- Finally allow users view balances

### Objective 
- Understand sUDT token mechanics on CKB
- Implement token creation and storage
- Understand the concept of state management via blockchain transactions
- Develop a working frontend with Next.js


### What i have learnt 
- Tokens in CKB do not reside as balances, but rather data stored in cells 
- Type script determines token mechanics (sUDT mechanics) 
- Transactions consume existing cells and produce new cells 
- Capacity (CKB) needed to save data in cells 
- The difference between CKB’s cell model and Ethereum’s account model
